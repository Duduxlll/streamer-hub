Versão: integração com site/iframe

O que esta versão prepara:
1. Remove o fluxo antigo de menu inicial como tela principal.
2. Mostra uma tela dentro do Godot: "Aguardando jogadores".
3. O site pode enviar jogadores para o Godot via postMessage.
4. O Godot mostra os jogadores carregados.
5. O botão "Iniciar corrida" fica dentro do Godot.
6. O site também pode iniciar a corrida por mensagem.
7. O Godot para a corrida quando completar o Top escolhido.
8. O Godot manda os vencedores de volta para o site.
9. Mantém velocidade 1x/2x/3x.
10. Mantém largada sem travar.
11. Melhora o painel lateral de ranking para nomes menores.

Mensagens que o site envia para o iframe:

Enviar jogadores:
{
  "type": "marble:init",
  "payload": {
    "players": ["Vitor", "Bing", "Jogador 003"],
    "top": 5
  }
}

Iniciar corrida:
{
  "type": "marble:start",
  "payload": {}
}

Alterar velocidade:
{
  "type": "marble:speed",
  "payload": { "value": 2 }
}

Resetar/voltar para espera:
{
  "type": "marble:reset",
  "payload": {}
}

Mensagens que o Godot devolve para o site:

Jogo pronto:
marble:ready

Jogadores carregados:
marble:loaded

Corrida iniciada:
marble:started

Cada colocado:
marble:place

Resultado final:
marble:finish

Também coloquei o arquivo:
EXEMPLO_SITE_IFRAME.html

Para testar:
1. Exporte o Godot para Web como index.html.
2. Copie EXEMPLO_SITE_IFRAME.html para a mesma pasta exportada.
3. Rode py -m http.server 5500.
4. Abra http://localhost:5500/EXEMPLO_SITE_IFRAME.html.
