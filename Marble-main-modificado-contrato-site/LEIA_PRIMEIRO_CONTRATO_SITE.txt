Versão: contrato do site

Esta versão suporta o contrato que seu site pediu:

Site -> Godot:
window.CORRIDA_DADOS_JSON = '{"jogadores":["fulano","ciclano"],"topN":5}'

Godot -> Site:
window.corridaResultado(["1º lugar", "2º lugar", ...])

O que foi adicionado:
1. scripts/Bridge.gd
2. Autoload Bridge no project.godot
3. main.gd lê Bridge.jogadores e Bridge.top_n ao iniciar.
4. Quando a corrida termina, Godot chama Bridge.enviar_resultado([...]).
5. Mantém o postMessage antigo também, então os dois modelos funcionam.
6. Arquivo de teste: EXEMPLO_CONTRATO_SITE.html.

Como testar:
1. Abra este projeto no Godot.
2. Aperte F5.
3. Se não estiver na Web, ele abre aguardando jogadores.
4. Exporte para Web como index.html.
5. Copie EXEMPLO_CONTRATO_SITE.html para a pasta exportada.
6. Rode py -m http.server 5500.
7. Abra http://localhost:5500/EXEMPLO_CONTRATO_SITE.html.

Importante sobre iframe:
Se o site usar iframe, a variável precisa estar no parent ou na própria janela do iframe.
Nesta versão o Bridge tenta ler:
- window.CORRIDA_DADOS_JSON
- window.parent.CORRIDA_DADOS_JSON

E no resultado tenta chamar:
- window.corridaResultado(...)
- window.parent.corridaResultado(...)
